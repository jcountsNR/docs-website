## Features
Collect and send telemetry data to New Relic from your Amazon Lattice services using our integration. Monitor your services, query incoming data, and build <InlinePopover type="dashboards" /> to observe everything at a glance. 

## Set up integration
The New Relic - Amazon VPC Lattice integartion uses [Amazon Kinesis Firehose](https://docs.aws.amazon.com/firehose/latest/dev/what-is-this-service.html) to send Logs to New Relic.

To set up monitoring For Amazon VPC Lattice:
  1. Navigate to the VPC feature in your [AWS console](https://console.aws.amazon.com/vpc)
  2. Select `Services` or `Service networks` under the `VPC Lattice` dropdown group, depending on which one you want to set up monitoring for.
  3. Select the Service or Service netowrk you wish to set up monitoring on
  4. Open the `Monitoring` tab and select `Edit monitoring settings`
  5. Toggle the `Access logs` option, and select the `Kinesis Data Firehose delivery stream` checkbox
  6. Select a Firehose Delivery Stream that points to New Relic from the dropdown, and click `Save changes`

<Callout variant="important">
To set up this integration, you must have a Kinesis Firehose delivery stream created. If you don't, you can create one by going to the Amazon Kinesis Firehose feature from your [AWS console](https://console.aws.amazon.com/firehose), and creating a delivery stream with `Destination: New Relic`.

[Learn more about creating an Amazon Kinesis Data Firehose delivery stream to New Relic.](https://docs.newrelic.com/docs/logs/forward-logs/stream-logs-using-kinesis-data-firehose/)
</Callout>

## Find and use data

To find your integration data: 

* For logs: Go to **[one.newrelic.com](https://one.newrelic.com) > Logs > Attributes > AWS** and select **logGroup** and/or **logStream**
